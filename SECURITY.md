# 🔒 Security Policy - HyperBoost v2.0

## ⚠️ Important Safety Notice

### Risk Levels Explained

| Level | Risk | Description |
|-------|------|-------------|
| 1 | 🟢 LOW | Safe settings changes only |
| 2 | 🟢 LOW | ADB shell optimizations |
| 3 | 🟡 MEDIUM | Root access required |
| 4 | 🟡 MEDIUM | PC + ADB optimizations |
| 5 | 🔴 HIGH | Root + PC maximum tweaks |
| OC | 🔴 CRITICAL | CPU/GPU overclocking |
| TH | 🔴 CRITICAL | Thermal unlock |

### Features That CAN Cause Damage
- CPU Overclock (modules/cpu_overclock.sh)
- GPU Overclock (modules/gpu_overclock.sh)
- Thermal Unlocker (modules/thermal_unlocker.sh)
- SELinux Bypass (exploits/sepolicy_injector.sh)
- Voltage Control (advanced)

### Safe Features
- Animation tweaks
- Cache cleaning
- Network optimization
- Debloating
- Game FPS settings
- Touch sensitivity
- ANGLE configuration
- Backup/Restore
